## Executive Summary
ECU Guardian is a smart vehicle monitoring and predictive maintenance application designed to provide drivers with real-time insights into their vehicle's health and performance. By connecting to a vehicle's OBD-II (On-Board Diagnostics) port, the system collects telemetry data from the Engine Control Unit and transforms raw sensor readings into actionable information.

The application serves four primary functions:

- **Driver Behavior Analysis:** Classifies driving patterns to help users understand and improve their driving habits
- **Fuel Efficiency Tracking:** Provides continuous, real-time mileage calculations
- **Fault Detection:** Identifies and translates diagnostic trouble codes into plain-language explanations
- **Predictive Maintenance:** Uses machine learning to anticipate potential failures before they occur
ECU Guardian empowers everyday drivers, fleet managers, and automotive professionals with the data they need to reduce maintenance costs, improve fuel economy, and prevent unexpected breakdowns.

### Real-Time Mileage Calculation
#### Overview
The Real-Time Mileage Calculation module provides continuous fuel efficiency monitoring by calculating miles per gallon (MPG) from live sensor data.

#### Formula
The system calculates fuel efficiency using the following formula:

```
MPG = (710.7 × VSS) / MAF
```
Where:

- **MPG** = Miles Per Gallon
- **VSS** = Vehicle Speed Sensor reading (mph)
- **MAF** = Mass Air Flow rate (g/s)
- **710.7** = Conversion constant accounting for fuel density and unit conversions
#### Input Parameters
| Parameter | Unit | Valid Range | Notes |

### Diagnostic Fault Detection
#### Overview
The Diagnostic Fault Detection module continuously scans for OBD-II Diagnostic Trouble Codes (DTCs) and translates them into human-readable explanations. This empowers drivers to understand vehicle issues without specialized knowledge.

#### Supported Diagnostic Modes
| Mode | Name | Description |
| ----- | ----- | ----- |
| **Mode 03** | Show Stored DTCs | Retrieves confirmed fault codes stored in ECU memory |
| **Mode 04** | Clear DTCs | Clears stored fault codes and resets monitors |
| **Mode 06** | Test Results | Returns on-board monitoring test results |
| **Mode 07** | Show Pending DTCs | Retrieves pending fault codes from current drive cycle |
#### Fault Code Structure
OBD-II fault codes follow a standardized format:

```
[System][Fault Type][Subsystem][Specific Fault]
P        0           1           23
```
**System Prefixes:**

| Prefix | System |
| ----- | ----- |
| P | Powertrain (engine, transmission) |
| B | Body (airbags, A/C, lighting) |
| C | Chassis (ABS, steering) |
| U | Network (communication) |
**Fault Type:**

| Digit | Meaning |
| ----- | ----- |
| 0 | Generic (SAE standard) |
| 1 | Manufacturer-specific |
#### Fault Code Lookup Table
The system maintains a comprehensive mapping table for code translation:

| Code | Description | Severity | Recommended Action |
| ----- | ----- | ----- | ----- |

| P0171 | System Too Lean (Bank 1) | Medium | [ACTION] |
| [CODE] | [DESCRIPTION] | [SEVERITY] | [ACTION] |
